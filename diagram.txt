DATA BUILD STAGE
================

Massive Flat Files (S3)
        |
        v
services/s3_client.py
        |
        v
pipeline/features.py
- load daily SPY minute bars
- convert timestamps to ET
- keep only 09:30 to 09:34
- compute per-day features
        |
        v
pipeline/dataset.py
- build spy_open_raw_minutes.csv
- build spy_open_features.csv
- apply deterministic label
        |
        v
trading_strategies/breakout_strategy.py
- momentum breakout rules
- produces ground-truth label: TAKE or PASS


OUTPUTS AFTER main.py
=====================

1. spy_open_raw_minutes.csv
- raw 1-minute opening bars
- 5 rows per trading day
- mainly for inspection / raw signal record

2. spy_open_features.csv
- one engineered row per trading day
- contains the ground-truth label
- used as retrieval memory and evaluation ground truth

RAG FLOW
========

One date from spy_open_raw_minutes.csv
        |
        v

- format the same 5 raw bars
        |
        +------------------------------+
        |                              |

rag/knowledge_base.py            spy_open_features.csv
- load strategy rules            - historical labeled examples
        |                              |
        +--------------+---------------+
                       |

- retrieve similar historical examples

                       |

  - current raw 5-bar sequence

  - retrieved historical examples

                       |

                       |
                       v
LLM
                       |
                       v
rag_results.csv
                       |
                       v
evaluation/evaluation.py
- compare predicted_label vs true_label in spy_open_features.csv


KEY IDEA
========

main.py only builds the datasets.

Ground truth comes from:
- trading_strategies/breakout_strategy.py
- stored in spy_open_features.csv as label

RAG happens after the dataset exists.

RAG does NOT create the label.
RAG uses:
- one date's raw 5-bar opening sequence
- strategy rules
- similar historical rows from other dates
- historical labels for those other dates
to help the LLM predict TAKE or PASS.

The current date must be excluded from retrieval.
